## 1. 什么是 CLI（大白话）

**CLI** = Command Line Interface = "命令行界面"，一句话：**用"一行一行打字下命令"的方式来指挥电脑。**

平时你用鼠标双击图标、点来点去，那叫**图形界面**。CLI 是另一条路：不点图标，直接打字告诉电脑你要干嘛。

**贴近生活的比喻 —— 点菜：**

- **图形界面** = 看着**图片菜单**用手指点。直观，但只能点菜单上有的。
- **CLI** = 直接走进厨房**跟厨师说话**："番茄切块、鸡蛋打散，盐少放。" 得先学会怎么开口（所以一开始有门槛），但学会后能精确指挥它做任何事。

**程序员为什么偏要打字？** 因为一句命令能办很多事、还能存成"脚本"批量自动跑，鼠标点击做不到。而且很多开发工具（比如 git）压根没有图形按钮，只有命令行版本——会打命令，等于拿到进开发世界的门票。

## 2. 怎么打开命令行

1. 按一下键盘左下角的 **Windows 键**（四个小方块图标）。
2. 直接打字 `powershell`。
3. 点出现的 **Windows PowerShell**。

弹出的深色窗口里，有一行像这样的字：

```
PS C:\Users\40315>
```

**怎么读这行：** 它表示"你现在站在 `C:\Users\40315` 这个文件夹里，可以打字了。" 其中 `40315` 是你的用户名，`C:\Users\40315` 是你的"家"（主目录）。最后的 `>` 后面闪动的光标，就是输入的地方。

## 3. 最重要的心理模型：你在文件夹里"走路"

命令行眼里，你的电脑就是**文件夹套着文件夹**，而你**永远站在其中某一个文件夹里**（这叫"当前目录"）。

```
C:\Users\你   ← 主目录（家）
│
├── 桌面 Desktop
├── 文档 Documents
└── 练习            ← 假设你现在站在这里
    └── note.txt
```

几乎所有操作，都是在回答三个问题：**我在哪？这里有什么？怎么走到别处？**

- `pwd` → 我在哪
- `ls` → 这里有什么
- `cd` → 怎么走

记住这个画面，命令行就不神秘了。

## 4. 核心命令逐个讲

### `pwd` —— 我在哪个文件夹？

- **大白话：** 告诉你当前站的完整路径。
- **来历：** **p**rint **w**orking **d**irectory（打印当前工作目录）。
- **例子：**
  ```
  pwd
  ```
  回显类似 `C:\Users\40315\练习`。迷路时第一个就打它。

### `ls` —— 这个文件夹里有什么？

- **大白话：** 列出当前文件夹里的所有文件和子文件夹。
- **来历：** **l**i**s**t（列表）。
- **例子：**
  ```
  ls
  ```
- **隐藏文件也要看：** 默认看不到隐藏项（比如 git 的 `.git`），加 `-Force`：
  ```
  ls -Force
  ```
- **看 `Mode` 列认类型：** `d` = 文件夹（directory），`a` = 普通文件（archive），`h` = 隐藏（hidden）。例如 `.git` 显示 `d--h--` 就是"隐藏的文件夹"。

### `cd` —— 走到另一个文件夹

- **大白话：** 换一个文件夹站。
- **来历：** **c**hange **d**irectory（改变目录）。
- **三种走法：**
  ```
  cd 练习                       # 相对路径：从当前位置往里走一层
  cd ..                         # 往外退一层（回到上级文件夹）
  cd C:\Users\40315\练习        # 绝对路径：不管在哪，直接跳过去
  cd ~                          # 回到家目录 C:\Users\你
  ```
- **走对了怎么确认？** 看提示符末尾。从 `...40315>` 进入练习后会变成 `...40315\练习>`。

> **绝对路径 vs 相对路径（重要）**
> - **绝对路径**：从盘符 `C:\` 写全，像报完整地址"北京市朝阳区xx路"，在哪都能到。
> - **相对路径**：只写"下一步"，像说"进隔壁房间"，只有知道你现在站哪才成立。
> - `cd 练习` 之所以有时报错"找不到"，是因为你当前位置旁边没有这个文件夹。先 `cd ~` 回家、或用 `pwd` 看清位置，再就近走，最稳。

### `mkdir` —— 新建一个文件夹

- **大白话：** 凭空造一个新文件夹。
- **来历：** **m**a**k**e **dir**ectory。
- **例子：**
  ```
  mkdir 练习
  ```
  回车后会列出新文件夹的信息（`d-----` 开头那行），就是成功了。

### `echo "内容" > 文件` —— 造文件并写入内容

- **大白话：** 把一句话存进一个文件里。
- **`>` 和 `>>` 的关键区别（务必记牢）：**
  | 符号 | 作用 | 后果 |
  |---|---|---|
  | `>` | **覆盖** | 把文件原有内容全删掉，换成新的 |
  | `>>` | **追加** | 在原内容**后面**接着写，原内容保留 |
- **例子：**
  ```
  echo "你好命令行" > note.txt        # 造文件 / 覆盖写入
  echo "这是第二行" >> note.txt       # 在后面追加一行
  ```
  回车后屏幕没反应是正常的（默默存好了）。

### `cat 文件` —— 把文件内容打出来看

- **大白话：** 把一个文件的内容整个打印到屏幕上。
- **来历：** **con**cat**enate**（拼接）。本来是把多个文件首尾相接拼成一串输出；只给一个文件时，就等于"原样打印这一个文件"，于是成了最常用的"快速看内容"命令。
- **例子：**
  ```
  cat note.txt
  ```

### `cls` —— 清屏

- **大白话：** 屏幕太乱时，一键清空。
- **来历：** **cl**ear **s**creen。
- **例子：**
  ```
  cls
  ```

## 5. 省力小技巧

- **Tab 键自动补全：** 文件夹/文件名打一半，按 **Tab**，自动补全剩下的，少打字也不怕拼错。
- **↑ 上箭头调历史：** 按 **↑**，调出刚打过的命令，不用重输。
- **运行你的 Python 程序：** `cd` 到 `.py` 文件所在的文件夹，然后：
  ```
  python 文件名.py
  ```

## 6. Win+R 和 CLI 的区别

- 你按 **Win+R** 弹出的"运行"小框，**本身不是 CLI**——它只是个"启动器"：你输一个程序名，它负责把那个程序打开，然后框就关了，不会接收命令、不会回你话。
- 但它能**帮你打开 CLI**：
  - 输 `cmd` → 打开**命令提示符**（老式黑框），是 CLI
  - 输 `powershell` → 打开 **PowerShell**，也是 CLI
  - 输 `command` → 是更老的 DOS 程序，现代 64 位 Windows 上一般已废弃，记 `cmd` 即可
- **判断标准：** 能让你**一行行持续打命令、每次都给反馈**的，才是 CLI；打一次就关、只负责"启动某个东西"的，是启动器（Win+R、桌面图标、开始菜单都属于这类）。

## 7. 常见报错怎么看

- **`PathNotFound` / `ObjectNotFound` / "找不到路径...因为该路径不存在"**
  → 十有八九是**文件名打错了**，或者**走错了文件夹**。
  → 自救：用 `pwd` 看自己在哪、用 `ls` 看这里到底有没有那个文件，再把名字打对重来。不可怕，改对就行。

## 8. 命令总结表（小抄）

| 命令 | 干啥的 | 怎么记 |
|---|---|---|
| `pwd` | 我现在在哪个文件夹 | **p**rint **w**orking **d**irectory |
| `ls` | 列出当前文件夹里有什么 | **l**i**s**t |
| `ls -Force` | 连隐藏的也列出来 | Force = 强制全显示 |
| `cd 名字` | 走进某文件夹 | **c**hange **d**irectory |
| `cd ..` | 往外退一层 | 两个点 = 上一层 |
| `cd ~` | 回到家目录 | ~ = 家 |
| `cd 完整路径` | 不管在哪直接跳过去 | 绝对路径 |
| `mkdir 名字` | 新建文件夹 | **m**a**k**e **dir**ectory |
| `echo "x" > 文件` | 造文件并写入（覆盖） | 一个 > = 覆盖 |
| `echo "x" >> 文件` | 在文件后面追加 | 两个 >> = 追加 |
| `cat 文件` | 把文件内容打出来看 | con**cat**enate |
| `cls` | 清屏 | **cl**ear **s**creen |
| `python 文件.py` | 运行 Python 程序 | —— |

**一句话总结：** CLI 就是"用打字跟电脑对话"。先用 `pwd`/`ls` 搞清"我在哪、有啥"，再用 `cd` 就近移动，剩下的都是在当前文件夹里增删改查。

## 9. 还没学 / 下一步

这份笔记覆盖了我们实际操作过的内容。以下是**还没学、但以后会用到**的常用命令，先记个名字，需要时再展开：

- **`cp` / `copy`** —— 复制文件
- **`mv` / `move`** —— 移动或**重命名**文件（学习地图里那个"把 starter.py 改名再自己重写"的技巧就用它）
- **`rm` / `del`** —— 删除文件、文件夹（有风险，要小心，单独学时再讲）
- **在编辑器里打开文件** —— 比如 `notepad note.txt`（记事本），或装了 VS Code 后用 `code .` 打开整个文件夹
- **通配符 `*`** —— 批量操作，比如 `ls *.txt` 只看所有 txt 文件
