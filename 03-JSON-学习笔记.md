## 1. 什么是 JSON（大白话）

你想记"一个人的信息"，平时会这样写：

```
姓名：Liz
年龄：18
城市：旧金山
```

JSON 就是把这种"一项一项的信息"，用电脑喜欢的标点符号包装一下：

```json
{
  "姓名": "Liz",
  "年龄": 18,
  "城市": "旧金山"
}
```

**内容一模一样，只是加了 `{ }`、`" "`、`:`、`,` 这几个符号。** 这就是 JSON。

- 全称是 JavaScript Object Notation，但**不用记英文**，记住它是"电脑用的信息记录格式"就够了。
- 它的最大特点：**人能读、电脑也能读**，所以成了网络上传数据的通用格式。

---

## 2. 5 个符号（认得就读懂）

| 符号 | 干嘛的 | 大白话 |
|---|---|---|
| `{ }` | 包住一整组信息 | 一张档案卡的边框 |
| `" "` | 包住文字 | "这是一段文字" |
| `:` | 连接"名称"和"值" | "姓名" **是** "Liz" |
| `,` | 隔开每一项 | "这项写完了，还有下一项" |
| `[ ]` | 装一串东西 | 一个清单 / 列表 |

**就这 5 个，认得它们，JSON 就读懂了。**

---

## 3. 两种基本结构

### 结构一：`{ }` 一组"名称: 值"（最常见，像档案卡）

```json
{
  "name": "Liz",
  "age": 18,
  "is_student": true
}
```

读法：这是一个人，名字 Liz、年龄 18、是不是学生 = 是。每项都是"**名称 : 值**"，中间冒号，项与项之间逗号。

### 结构二：`[ ]` 一串清单（像列表）

```json
["苹果", "香蕉", "橘子"]
```

读法：一个水果清单，三样东西，逗号隔开。

> ⚠️ **关键区别（最容易混）：**
> - **`{ }` 里面**才有"名称: 值"。
> - **`[ ]` 里面**只有一个挨一个的**值**，**不带名称**。
> - 错误示范：`["hobbies": "唱歌"]` ❌ —— 方括号里不能放名称。
> - 如果想给清单起名，要用 `{ }` 包起来：`{ "hobbies": ["唱歌"] }` ✅

---

## 4. 值可以是哪几种

冒号右边的"值"，就这几类：

| 类型 | 写法 | 要不要引号 |
|---|---|---|
| 文字 | `"Liz"`、`"旧金山"` | **要**双引号 |
| 数字 | `18`、`3.14` | **不要**引号 |
| 是 / 否 | `true` / `false` | 不要引号 |
| 清单 | `["a", "b"]` | 用方括号 |
| 套娃 | 一个完整的 `{ }` | 见第 5 节 |

**口诀：文字要引号，数字不用，是/否写 `true` / `false`。**

---

## 5. 套娃：嵌套

现实信息没那么扁平，JSON 能像俄罗斯套娃一样层层嵌套——**值的里面，还能再装一个 `{ }` 或 `[ ]`**。

```json
{
  "name": "Liz",
  "age": 18,
  "hobbies": ["学AI", "唱歌"],
  "account": {
    "github": "Liz-Ji",
    "notes_count": 3
  }
}
```

逐行读：
- `name`、`age` —— 两个简单的项；
- `hobbies` —— 它的值是个**清单 `[ ]`**，装了两个爱好；
- `account` —— 它的值又是一个**完整的 `{ }`**（套娃！），里面再装 `github` 和 `notes_count`。

**这就是 JSON 最有用的地方：复杂信息一层套一层，结构清清楚楚。**

---

## 6. 怎么读一段 JSON

看到一大坨 JSON 别慌，问自己两个问题：

1. **最外面是 `{` 还是 `[`？** → `{` 是"一组属性"，`[` 是"一个清单"。
2. **顺着冒号读**："谁(名称) 是 啥(值)"；遇到 `{` 或 `[` 就知道"这里要往里套一层"。

---

## 7. 踩过的坑（很值钱）

这些是写 JSON 最常见的错，我都亲手踩过、改对了：

| 坑 | 错误示范 | 正确 |
|---|---|---|
| 用了单引号 | `'name': '珍珠奶茶'` | `"name": "珍珠奶茶"`（**只能双引号**）|
| 漏了逗号 | 两项之间没逗号 | 每项后加 `,`，**最后一项不加** |
| 方括号里放名称 | `["hobbies": "唱歌"]` | 名称只能在 `{ }` 里 |
| 套娃逗号判断错 | 内层最后一项加了逗号 | **每一层各管各的，最后一项都不加** |
| 数字加了引号 | `"age": "18"` | 数字不要引号：`"age": 18` |

> 💡 **逗号黄金规律（一句话记牢）：**
> **在每一层 `{ }` 或 `[ ]` 内部，"还有下一项"就加逗号，"我是这一层最后一项"就不加。** 每一层各算各的。

> 💡 **看编辑器找错：** 在 VS Code 等编辑器里写 JSON，格式不对会出现**红/黄波浪线**提醒。波浪线消失 = 格式干净。

## 8. 为什么学 AI 必须懂它

**你给 AI 发请求、AI 回你的回答，传来传去的数据，全都是 JSON 格式。**

比如你问 AI 一句话，它返回的东西大概长这样：

```json
{
  "content": "你好！有什么可以帮你的？",
  "model": "claude",
  "tokens_used": 12
}
```

- `content` 那一项 = AI 的回答
- `tokens_used` = 这次用了多少 token

**不先认识 JSON，看 AI 的返回就是一团符号；认识了，一眼就能找到"回答在哪、用了多少 token"。** 这就是它是 API 地基的原因——下一块（API）马上用得上。

## 9. 练习与答案

### 练习 1：描述一杯奶茶
名称 珍珠奶茶、价格 18、加冰 是。

```json
{
  "name": "珍珠奶茶",
  "price": 18,
  "ice": true
}
```

### 练习 2：三个爱好（清单）

```json
["唱歌", "学AI", "练字"]
```
> 注意：纯清单不需要名称，最外面直接 `[ ]`。

### 练习 3：描述你自己（套娃）

```json
{
  "name": "Liz",
  "age": 18,
  "hobbies": ["学AI", "唱歌"],
  "account": {
    "github": "Liz-Ji",
    "notes_count": 3
  }
}
```
> 难点：`hobbies` 值是 `[ ]`，`account` 值是套娃的 `{ }`；逗号每层各管各的，最后一项都不加。

## 10. 小抄总结

| 要点 | 内容 |
|---|---|
| JSON 是啥 | 电脑用的"档案卡"，记录信息的格式 |
| `{ }` | 一组"名称: 值" |
| `[ ]` | 一个清单（只装值，不装名称）|
| `:` | 连接名称和值 |
| `,` | 隔开各项，**最后一项不加** |
| `" "` | 文字两边要双引号；数字不用；是/否用 `true`/`false` |
| 套娃 | 值里可以再装 `{ }` 或 `[ ]`，一层套一层 |
| 读法 | 顺着冒号念"谁 是 啥"，遇 `{`/`[` 往里套一层 |

**一句话总结：JSON = 名称:值，用几个符号包起来，还能套娃。看懂它 = 看懂 AI 给你回了啥。**
