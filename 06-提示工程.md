# 06 · 提示工程（Prompt Engineering）

> Stage 2 学习笔记 ｜ 状态：✅ 已毕业
> 一句话总结：**提示工程 = 把给 AI 的「点菜单」写清楚的手艺，让它端出你真正想要的菜，而不是它瞎猜的。**

---

## 一、核心心法（最重要，记这一条就够）

> **AI 不会读心。你给的信息越具体，它越不用猜，端出来的越对。**
>
> 推论：**凡是你心里已经有答案的（比喻、字数、风格、格式），别让 AI 猜，直接写进 prompt。**

比喻：
- LLM = 厨房里的接龙高手
- prompt（提示词）= 你递进厨房的那张「点菜单」
- 提示工程 = 把这张点菜单写清楚的手艺

---

## 二、点菜单的 4 个基本零件

写 prompt 时，尽量把这几样填上，越具体越好：

| 零件 | 说明 | 例子 |
|---|---|---|
| 🎯 任务 | 让它干啥 | 写小红书文案 |
| 👤 受众 | 写给谁看 | 0 基础文科生小白 |
| 🎨 风格/角色 | 什么口吻 | 大白话、像聊天 |
| 📐 格式/要求 | 长度、结构、emoji… | 100 字内、钩子开头、结尾引导关注 |

---

## 三、四大招式（每招都写代码验证过）

### 招式 1：角色设定（system prompt）🎭

**是什么**：在提问前先告诉 AI「你是谁」，它就会切换到那个身份的口吻和水平。
**比喻**：同一个问题，问菜场大爷 vs 问大学教授，答案天差地别。
**心法**：想让 AI 怎么说话，就先告诉它它是谁。

**验证结果**（practice_1_role.py）：同一个问题「什么是租赁合约」，喂 4 个角色——
- 律师 → 引用《民法典》第几条，一本正经
- 幼儿园老师 → 「就像借玩具玩」，打比方
- JSON 机器 → 只吐 JSON，不说人话
- 东北大爷 → 「哎呦喂，这不跟处对象似的嘛」😂

→ 角色一换，回答的深浅、口吻、格式全变。✅

---

### 招式 2：few-shot 给例子 🖼️

**是什么**：与其用形容词描述要什么，不如直接甩 1~3 个「成品样品」给 AI 照着模仿。
**比喻**：找裁缝做衣服，与其说「要温柔点的」，不如直接掏张照片「照这件做」。
**心法**：例子 = 把「只可意会」变成「照着抄」。描述靠猜，例子靠抄。

**验证结果**（practice_2_fewshot.py）：情绪分类任务（正面/负面/中性）
- 第一回合（不给例子，0-shot）：**4 / 5**（「电影一般但特效还行」被判成正面 ❌）
- 第二回合（给 3 个例子，few-shot）：**5 / 5**（同一道题被救回，判对为中性 ✅）

→ 给例子，准确率实打实从 4/5 提到 5/5。✅

---

### 招式 3：CoT 一步一步想（Chain-of-Thought）🧮

**是什么**：复杂/绕弯的题，加一句「请一步一步想」，AI 会把步骤摊开，答得更准。
**比喻**：别让 AI 口算，让它列竖式。
**使用时机**：简单题别加（浪费）；要「算来算去、判断来判断去」的题一定加。

**验证结果**（practice_3_cot.py）：一道绕弯的苹果应用题
- 直接问（逼它报数字）：答 **3** ❌
- 请它一步步想：列 5 步竖式，答 **2** ✅

→ 同一道题，加一句「一步步想」，从错变对。✅

---

### 招式 4：迭代打磨（Iterative Refinement）✏️

**是什么**：prompt 不是一次写好的，是改出来的。第一版模糊 → 看效果 → 改具体 → 再看 → 再改。
**心法**：prompt 写得越具体，AI 越不用猜，产出越对。

**验证结果**（practice_4_refine.py）：同一个任务「写 token 文案」，3 版 prompt 由粗到细
- 第 1 版「写一条关于 token 的文案」→ 瞎猜成币圈文案，没法用
- 第 2 版「小红书、通俗易懂」→ 好些，但跑偏成「登录令牌」
- 第 3 版（角色+积木块比喻+钩子+字数+emoji）→ 命中「AI 文字单位」，基本能直接发 ✅

→ prompt 越改越具体，产出越来越能用。✅

## 四、代码骨架（复用模板）

```python
from openai import OpenAI

client = OpenAI(
    api_key="你的key（建议改用环境变量，别写死）",
    base_url="https://api.deepseek.com",   # 指向 DeepSeek
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "角色设定"},   # 招式1
        {"role": "user", "content": "示例问题"},      # few-shot 示例(招式2)
        {"role": "assistant", "content": "示例答案"},
        {"role": "user", "content": "真正的问题"},
    ],
)
print(response.choices[0].message.content)
```

**三种身份记牢**：
- `system` = 角色设定
- `user` = 用户说的话
- `assistant` = AI 的回答（few-shot 里用它伪造范例）

## 五、踩过的坑（血泪记录）

| 坑 | 现象 | 解决 |
|---|---|---|
| 标点中英文混用 | 中文逗号/冒号电脑不认 | 全用英文半角 |
| 字典/参数符号混 | `'role' = 'x'` 写成等号 | 字典内部用冒号 `:`，参数赋值才用 `=` |
| 漏逗号 | `Perhaps you forgot a comma?` | 括号里参数之间、字典之间都要逗号 |
| 拼写少字母 | `choice`→应是 `choices`、`message`→`messages` | 固定写法别背，回去抄对的 |
| 模型名写错 | `deepseek_chat` | 是横杠 `deepseek-chat` |
| 模块找不到 | `No module named 'openai'` | openai 装在 3.13，但用 3.14 跑了 → 多环境打架 |
| 多环境打架 | 装的 Python 和跑的不是同一个 | 用 `py -3.13 文件名.py` 点名版本运行 |

**读报错口诀**：从下往上读 → 先看最后一行（错误类型）→ 再看 line 几（在哪）。报错是助手，不是敌人。

**运行命令**：`py -3.13 文件名.py`（点名用装了 openai 的 3.13）

## 六、重要概念澄清

- **提示 ≠ 训练模型**：训练是改 AI 的大脑（大厂干，贵、慢）；提示是每次临时交代任务（免费、瞬间，AI 大脑没变，挂了就忘）。我们做的全是「提示」。
- **OpenAI 包 ≠ OpenAI 公司**：`from openai import OpenAI` 里的 OpenAI 是个「通用电话机」，拨 DeepSeek 的号就打给 DeepSeek（因为 DeepSeek 兼容 OpenAI 接口，像 USB 通用口）。
- **Python vs JSON**：Python 是干活的厨师；JSON 是装数据的打包盒。自己肚子里现用 → Python 字典；要存要传 → JSON。
