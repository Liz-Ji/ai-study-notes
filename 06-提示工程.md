## 一、核心心法（最重要，记这一条就够）

> **AI 不会读心。你给的信息越具体，它越不用猜，端出来的越对。**

推论（贯穿全篇的口诀）：
> **凡是你心里已经有答案的（比喻、字数、风格、格式），别让 AI 猜，直接写进 prompt。**

比喻体系（和前面笔记一致）：
- LLM = 厨房里的接龙高手
- prompt（提示词）= 你递进厨房的那张「点菜单」
- 提示工程 = 把这张点菜单写清楚的手艺

**反面例子**：只说「帮我写一条小红书文案」→ AI 瞎猜成「护肤换季烂脸自救」，跟你八竿子打不着。
**正面例子**：把主题、受众、风格、格式都写清楚 → AI 端出基本能直接发的成品。

---

## 二、点菜单的 4 个基本零件

写任何 prompt 前，尽量把这 4 样填上，越具体越好：

| 零件 | 它回答什么问题 | 反面（太空） | 正面（够细） |
|---|---|---|---|
| 🎯 任务 | 让 AI 干啥？ | 「写点东西」 | 「写一条小红书文案讲什么是 token」 |
| 👤 受众 | 写给谁看？ | （不写） | 「0 基础、学 AI 的文科生小白」 |
| 🎨 风格/角色 | 什么口吻？ | （不写） | 「大白话、像跟朋友聊天」 |
| 📐 格式/要求 | 长度/结构/形式？ | （不写） | 「100 字内、钩子开头、结尾引导关注、带 3 个 emoji」 |

> 自检小技巧：写完一个 prompt，问自己「这 4 个零件，AI 还需要猜哪个？」——凡是要它猜的，补上。

### 招式 1 · 角色设定（system prompt）🎭

**是什么**：在提问前先告诉 AI「你是谁」，它会自动切换到那个身份的口吻、深浅和格式。
**比喻**：同一个问题「啥是租房合同」，问菜场大爷 vs 问大学教授，答案天差地别。区别不在问题，在「回答的人」。
**心法**：**想让 AI 怎么说话，就先告诉它它是谁。** 角色 = 一次性搞定「风格 + 受众 + 深浅」的偷懒神器。

**核心代码思路**：用一本「角色字典」装好多个身份，循环着挨个问。

```python
roles = {
    "严肃律师": "你是一位严谨的合约律师，请用专业、正式的语气回答，可以引用相关条款。",
    "幼儿园老师": "你是一位幼儿园老师，请用最简单、亲切的大白话解释，就像讲给小朋友听。",
    "JSON机器": "你是一个只输出JSON的机器，请用JSON格式回答，包含definition和example两个字段，不要多余的话。",
    "东北唠嗑大爷": "你是一个东北唠嗑大爷，请用最幽默、接地气的语气回答，最好带点儿东北俗语。",
}

question = "请帮我解释什么是租赁合约"

for name, setting in roles.items():            # 把字典里的角色一个个翻出来
    print("=" * 30)
    print(f"【{name}】的回答：")
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": setting},    # ← 角色设定塞这里
            {"role": "user", "content": question},
        ],
    )
    print(response.choices[0].message.content)
```

**验证结果**：同一个问题，4 个角色答得完全不同——
- 律师 → 引《民法典》第703、705条，一本正经
- 幼儿园老师 → 「就像借玩具玩」，打比方哄小孩
- JSON 机器 → 只吐 `{"definition":..., "example":...}`
- 东北大爷 → 「哎呦喂，这不跟处对象似的嘛」😂

→ **结论：角色一换，回答的深浅、口吻、格式全跟着变。** ✅

### 招式 2 · few-shot 给例子 🖼️

**是什么**：与其用一堆形容词描述「我要什么」，不如直接甩 1~3 个「成品样品」给 AI 照着模仿。
**比喻**：找裁缝做衣服，与其说「要温柔点的」，不如掏张照片「照这件做」。
**心法**：**例子 = 把「只可意会」变成「照着抄」。** 描述靠猜，例子靠抄。
**关键**：示例要塞进 `messages`，用 `user`（假装提问）+ `assistant`（假装 AI 的标准答案）一组组打样，放在真问题前面。

**核心代码思路**：A/B 对比——先「裸考」（不给例子），再「给 3 个例子」，用准确率说话。

```python
# 考题本：句子 → 标准答案（标准答案攥在我们手里，不给 AI 看！）
test_data = {
    "这家店的服务太棒了，我超满意！": "正面",
    "等了一个小时菜还没上，气死了": "负面",
    "今天周三，我下午三点有个会": "中性",
    "这电影剧情一般，但特效还行": "中性",
    "客服态度恶劣，再也不来了": "负面",
}

# —— 第二回合：给 3 个例子 ——
correct2 = 0
for sentence, answer in test_data.items():
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "user", "content": "判断情绪，只回答正面/负面/中性：饭菜很好吃"},
            {"role": "assistant", "content": "正面"},
            {"role": "user", "content": "判断情绪，只回答正面/负面/中性：东西一般般，价格还行"},
            {"role": "assistant", "content": "中性"},        # ← 故意示范"模棱两可=中性"
            {"role": "user", "content": "判断情绪，只回答正面/负面/中性：态度差到爆"},
            {"role": "assistant", "content": "负面"},
            {"role": "user", "content": f"判断情绪，只回答正面/负面/中性：{sentence}"},  # ← 真问题
        ],
    )
    reply = response.choices[0].message.content.strip()     # strip 擦掉前后空格，免得冤枉答错
    if reply == answer:                                     # 跟标准答案对一对
        correct2 = correct2 + 1                             # 对了，计分器 +1
print(f"得分：{correct2} / {len(test_data)}")
```

**验证结果**：
- 第一回合（0-shot，不给例子）：**4 / 5**（「电影一般但特效还行」被判成正面 ❌）
- 第二回合（few-shot，给 3 个例子）：**5 / 5**（那道模棱两可题被救回，判对为中性 ✅）

→ **结论：给例子，准确率实打实从 4/5 提到 5/5。** 而且故意在例子里塞「一般般=中性」，精准救回了那道错题。✅

### 招式 3 · CoT 一步一步想（Chain-of-Thought）🧮

**是什么**：复杂/绕弯的题，加一句「请一步一步想」，AI 会把思考步骤摊开，答得更准。
**比喻**：别让 AI 口算，让它列竖式。口算容易拍脑袋错；列竖式慢一点但稳。
**使用时机**：简单题（如翻译一句话）别加，浪费；要「算来算去、判断来判断去」的绕弯题，一定加。

**核心代码思路**：同一道绕弯数学题，「直接问」vs「请一步步想」，对比对错。

```python
question = "小明有24个苹果，先给小红三分之一，又给小刚6个，自己吃了2个，剩下平均分给4个朋友，每人几个？"
# 标准答案：24÷3=8；24-8-6-2=8；8÷4=2 → 2

# 第一回合：逼它直接报数字
response1 = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": f"{question}\n请直接告诉我最后的数字答案，不要解释。"}],
)
print(response1.choices[0].message.content)

# 第二回合：请它一步一步算
response2 = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": f"{question}\n请一步一步地计算，列出每一步，最后给出答案。"}],
)
print(response2.choices[0].message.content)
```

**验证结果**：
- 直接问（口算）：答 **3** ❌
- 请它一步步想：列 5 步竖式（24÷3=8 → 24-8=16 → 16-6=10 → 10-2=8 → 8÷4=2），答 **2** ✅

→ **结论：同一道题，加一句「一步步想」，从错变对。** ✅

### 招式 4 · 迭代打磨（Iterative Refinement）✏️

**是什么**：prompt 不是一次写好的，是改出来的。第一版模糊 → 看效果 → 改具体 → 再看 → 再改。
**心法**：prompt 写得越具体，AI 越不用猜，产出越对。（其实你改小红书标题、debug，都是在迭代打磨。）

**核心代码思路**：同一个任务，3 版由粗到细的 prompt（用「列表」排队装），循环跑出来对比。

```python
prompts = [                                    # 列表 [ ]：排队装，不用配对
    "写一条关于token的文案",                                              # 第1版：超粗
    "写一条小红书文案，介绍什么是token，要通俗易懂",                        # 第2版：细一点
    "你是大白话科普博主，给0基础文科生写一条小红书文案讲什么是token。"
    "用'积木块'比喻；开头一句钩子，结尾引导关注；100字内，带2个emoji。",    # 第3版：全配齐
]

for i in range(len(prompts)):                  # range(3) 生成 0,1,2，循环三遍
    print("=" * 40)
    print(f"第 {i+1} 版 prompt：{prompts[i]}")  # i+1 让人看着是"第1版"（电脑从0数）
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": prompts[i]}],
    )
    print(response.choices[0].message.content)
```

**验证结果**（同一任务「讲 token」）：
- 第 1 版「写一条关于 token 的文案」→ 瞎猜成**币圈**文案（通往未来的钥匙…），没法用
- 第 2 版「小红书、通俗易懂」→ 好些，但跑偏成**登录令牌/防盗号**那个 token
- 第 3 版（角色+积木块比喻+钩子+引导关注+字数+emoji）→ 命中**AI 文字单位**，基本能直接发 ✅

→ **结论：prompt 越改越具体，产出越来越对、越能用。** ✅

## 四、可复用的代码骨架（以后直接抄这个改）

```python
from openai import OpenAI                    # 从工具箱拿出"电话机"

client = OpenAI(                             # 造电话机，配 DeepSeek（换家只改3处）
    api_key="你的key",                        # 处①凭什么打（建议改环境变量，别写死）
    base_url="https://api.deepseek.com",     # 处②拨什么号
)

response = client.chat.completions.create(   # 拨号、说话
    model="deepseek-chat",                   # 处③找哪位接
    messages=[
        {"role": "system", "content": "角色设定"},     # 招式1
        {"role": "user", "content": "示例问题"},        # 招式2 few-shot 示例
        {"role": "assistant", "content": "示例答案"},
        {"role": "user", "content": "真正的问题"},
    ],
)
print(response.choices[0].message.content)   # 从回信里掏出正文（固定写法，照抄）
```

**三种身份记牢**：`system`=角色设定 ｜ `user`=用户说的话 ｜ `assistant`=AI 的回答（few-shot 里用它伪造范例）。

**`client.chat.completions.create` 怎么读**：点 `.` 读成「的」；`.create` 是「拨打」这个动作；括号里是通话内容；`response` 是装回信的盒子。

## 五、踩过的坑（血泪记录，含解决办法）

| 坑 | 报错/现象 | 原因 | 解决 |
|---|---|---|---|
| 标点中英混用 | 各种 invalid syntax | 中文逗号/冒号电脑不认 | 全用英文半角 |
| 冒号写成等号 | invalid syntax | 字典里键值该用冒号 | 字典内部用 `:`；参数赋值才用 `=` |
| 漏逗号 | `Perhaps you forgot a comma?` | 括号里参数之间、字典之间没逗号 | 都补上逗号 |
| 拼写少字母 | `AttributeError` / 黄线 | `choice`→`choices`、`message`→`messages` | 固定写法别背，回去抄对的 |
| 变量名记岔 | 黄色波浪线 | `right`→应是 `reply`、`sentences`→`sentence` | 顺着黄线找名字对不对 |
| 模型名写错 | 模型找不到 | `deepseek_chat`（下划线） | 是横杠 `deepseek-chat` |
| 模块找不到 | `No module named 'openai'` | openai 装在 Python 3.13，却用 3.14 跑 | 多环境打架 ↓ |
| 多环境打架 | 同上 | 电脑装了多个 Python 版本 | 用 `py -3.13 文件名.py` 点名版本运行 |

**读报错口诀**：从下往上读 →（1）最后一行看错误类型 →（2）看 `line 几` 定位 →（3）很多时候它直接给你提示（如"忘了逗号？"）。**报错是助手，不是敌人。**
**黄线 vs 红线**：黄色波浪线 = 提醒（如"导入没用上""名字不认识"）；红线/红字才是真错误。
**运行命令**：`py -3.13 文件名.py`

## 六、重要概念澄清（容易混的）

- **提示 ≠ 训练模型**：训练是改 AI 的大脑（大厂干，烧钱烧时间，永久改变）；提示是每次临时交代任务（免费、瞬间，AI 大脑没变，挂电话就忘）。我们做的全是「提示」。打分、判断对错，是「我们」在改卷，跟训练 AI 无关。
- **OpenAI 包 ≠ OpenAI 公司**：`from openai import OpenAI` 里的 OpenAI 是个「通用电话机」，拨 DeepSeek 的号就打给 DeepSeek。因为 DeepSeek 兼容 OpenAI 接口（像 USB 通用口），所以能借这套写法去用它——这就是「换家只改 3 处」。
- **Python vs JSON**：Python 是干活的厨师（执行动作）；JSON 是装数据的打包盒（存/传数据）。自己肚子里现用 → Python 字典 `{}`；要存盘或传给别的程序 → 打包成 JSON。
- **字典 `{}` vs 列表 `[]`**：字典是「通讯录」（名字配内容，成对）；列表是「排队名单」（一串，挨个排，不配对）。
